# Specs

This change is a pure architectural refactoring — converting the existing MVVM View/ViewModel layer to TCA features while preserving every existing behavior.

- **New capabilities**: None. No new user-facing features or behaviors are introduced.
- **Modified capabilities**: None. All existing specs (`instant-share-secure-discovery-trust`, etc.) describe external behavior and API contracts that remain unchanged.

No spec files are needed because the refactoring only changes HOW the code is structured internally, not WHAT it does.
